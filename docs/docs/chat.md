# Chat


`alpha_app.chat.Chat` is the central abstraction. Each conversation is a Chat object that owns a Claude subprocess, a message list, a state machine, and persistence to Postgres.

## Construction

### `Chat(*, id: str, claude: Claude | None = None)`

Create a new Chat. If `claude` is provided, starts in READY state; otherwise starts COLD.

- `id` — a 12-character URL-safe nanoid, generated by `generate_chat_id()`
- `claude` — optional pre-existing Claude subprocess

### `Chat.from_db(chat_id, created_at, updated_at, data) -> Chat` *classmethod*

Restore a Chat from a Postgres row. Born COLD (no subprocess). Restores session UUID, title, token counts, injected topics, human turn counter, and the recall seen-cache from the JSONB `data` column.

### `Chat.clone() -> Chat`

Create a fork-ready copy. Returns a new Chat with a fresh ID that, when its Claude subprocess starts, will use `--fork-session` to branch from the original session. The original is untouched. The clone is born COLD and headless — no broadcast, no WebSocket, no sidebar presence.

Used by Dusk to create ghosts that write day capsules.

## Instance attributes

| Attribute | Type | Description |
|-----------|------|-------------|
| `id` | `str` | Chat ID (nanoid, 12 chars) |
| `session_uuid` | `str \| None` | Claude Code session UUID, captured from first `ResultEvent` |
| `state` | `ConversationState` | Current conversation state |
| `suggest` | `SuggestState` | Current suggest pipeline state |
| `title` | `str` | Chat title (first user message, truncated to 80 chars) |
| `created_at` | `float` | Unix timestamp of creation |
| `updated_at` | `float` | Unix timestamp of last activity |
| `messages` | `list[UserMessage \| AssistantMessage \| SystemMessage]` | The canonical conversation record |
| `on_broadcast` | `Callable \| None` | WebSocket broadcast callback, set by ws.py |
| `on_reap` | `Callable \| None` | Post-reap callback for state change broadcast |

## State machine

Chat has a 2D state vector with two orthogonal dimensions.

### ConversationState

| State | Wire value | Meaning |
|-------|-----------|---------|
| `STARTING` | `"starting"` | Subprocess launching |
| `READY` | `"idle"` | Idle, waiting for input |
| `RESPONDING` | `"busy"` | Claude generating a response |
| `COLD` | `"dead"` | No subprocess |

### SuggestState

| State | Meaning |
|-------|---------|
| `DISARMED` | No post-turn suggest pending |
| `ARMED` | Will consider firing suggest after the current turn |
| `FIRING` | Suggest pipeline is running |

## Turns

### `async turn() -> AsyncContextManager[Turn]`

Acquire an exclusive turn on this Chat. Only one Turn can be active at a time. Auto-starts Claude if COLD.

```python
async with await chat.turn() as t:
    await t.send(user_message)       # UserMessage — full persistence + broadcast
    await t.send(content_blocks)     # Raw list[dict] — no enrichment, no persistence
    response = await t.response()    # Wait for completion, return AssistantMessage
```

The turn lock is released when `ResultEvent` arrives from Claude, or when the context manager exits.

### `Turn.send(msg_or_content)`

Send a message within a turn. Two modes:

- **`UserMessage`** — persists to `app.messages`, broadcasts `user-message` to WebSocket, extracts title from first message, resets output token counter, sends content blocks to Claude.
- **Raw content blocks** (`list[dict]`) — sends directly to Claude with no enrichment, no persistence, no broadcast. Used for steering messages (capsule prompts, narration, etc.).

### `Turn.response() -> AssistantMessage | None`

Wait for Claude to finish the current turn. Returns the completed `AssistantMessage` from `messages[-1]`, or `None` if no response was generated.

### `async interject(content: list[dict])`

Fire-and-forget message that bypasses the turn lock. No response tracking. Used for alarms, nudges, and approach lights. Auto-starts Claude if COLD.

## Lifecycle

### `async _ensure_claude()`

Auto-start Claude if not alive. Creates a fresh Claude subprocess with:
- Current system prompt (stored on `_system_prompt`)
- Fresh MCP server (`create_alpha_server`)
- Model from `constants.CLAUDE_MODEL`
- `--resume` if `session_uuid` exists; `--fork-session` if `_fork_from` is set

This is the CHAT-V2 auto-start primitive. Callers never check `is_alive` themselves.

### `async wait_until_ready() -> AssistantMessage | None`

Wait for Claude to finish current work and return the response. The universal "ask the duck, wait for the duck, read what the duck said" primitive. Used by headless jobs (Dawn, Dusk, Solitude) to sequence steps.

### `async reap()`

Kill the subprocess. State → COLD, suggest → DISARMED. Snapshots token state before killing so it survives for the sidebar.

### `async interrupt()`

Interrupt the current turn. Delegates to `reap()`.

### `async _on_claude_reap()`

Callback invoked by Claude's internal reap timer when it stops itself after 60 minutes of inactivity. Chat updates its own state and broadcasts the change.

## Persistence

### `async flush() -> int`

Write dirty messages to Postgres. Returns count written. UPSERT by `(chat_id, ordinal)` where ordinal is the array index in `messages[]`. Also persists chat metadata (token counts, title, session UUID, seen IDs, turn counter) via `persist_chat()`.

This is the ONLY write path for messages. Dawn, Solitude, WebSocket turns — all go through here.

### `async load_messages()`

Load messages from `app.messages` into `messages[]`. Called after `from_db()` to hydrate the conversation. Messages are loaded eagerly so ordinals match Postgres from the start.

### `to_data() -> dict`

Serialize chat metadata as a JSONB-ready dict. Includes session UUID, title, token counts, injected topics, seen IDs, and human turn counter.

### `messages_to_wire() -> list[dict]`

Serialize all messages for the "gimme the fucking chat" payload sent on `join-chat`.

## Token state

Properties that delegate to the underlying Claude subprocess, with fallback to cached values when the subprocess is dead.

| Property | Type | Description |
|----------|------|-------------|
| `token_count` | `int` | Current context window usage |
| `context_window` | `int` | Total context window size |
| `input_tokens` | `int` | Input tokens (current turn) |
| `total_input_tokens` | `int` | OTel-compliant: uncached + cache_creation + cache_read |
| `cache_creation_tokens` | `int` | Prompt cache creation tokens |
| `cache_read_tokens` | `int` | Prompt cache read tokens |
| `output_tokens` | `int` | Output tokens (current turn) |
| `stop_reason` | `str \| None` | Why Claude stopped |
| `response_model` | `str \| None` | Model that generated the response |
| `response_id` | `str \| None` | Response ID from the API |
| `usage_5h` | `float \| None` | 5-hour quota usage (from proxy) |
| `usage_7d` | `float \| None` | 7-day quota usage (from proxy) |

### `reset_output_tokens()`

Reset the output token accumulator. Called at turn start.

### `pop_api_error() -> dict | None`

Return and clear the last API error captured by the proxy.

## WebSocket state

### `wire_state(**overrides) -> dict`

Build the `chat-state` payload for WebSocket broadcasts. Includes state, title, timestamps, session UUID, token counts, context window, and topic states (if a topic registry is available).

## Event dispatch

Events from Claude flow through `_on_claude_event()`, which dispatches to focused handler methods via a type → method-name dict:

| Event type | Handler | Bookshelf | Broadcast |
|-----------|---------|-----------|-----------|
| `StreamEvent` | `_handle_stream` | Accumulate text/thinking/tool deltas on AssistantMessage | `text-delta`, `thinking-delta`, `tool-use-start`, `tool-use-delta` |
| `AssistantEvent` | `_handle_assistant` | Accumulate tool_use blocks | `tool-call` |
| `UserEvent` | `_handle_user_echo` | Pencil→ink confirmation, tool result updates | `user-message` (confirmed), `tool-result` |
| `ResultEvent` | `_handle_result` | Finalize AssistantMessage with metadata, flush, close Logfire span, fire suggest | `assistant-message`, `chat-state`, `exception` (truncation/API error) |
| `SystemEvent` | `_handle_system` | Reset orientation/topics/seen on compact_boundary; create SystemMessage for task notifications | `agent-started`, `agent-progress`, `agent-done`, `system-message` |
| `ErrorEvent` | `_handle_error` | Nothing | `error` |

### Spontaneous response detection

If a `StreamEvent` or `AssistantEvent` arrives while the chat is READY (no active turn), the chat transitions to RESPONDING and opens a Logfire span. This handles background tasks and system-initiated messages.

## Post-turn suggest

### `async _post_turn_suggest(user_text, assistant_text)`

After a human-initiated turn completes, the suggest pipeline sends a `<system-reminder>` prompting Alpha to reflect on the exchange and store anything worth remembering. Fires on an N=3 cadence (turns 1, 4, 7, 10, ...). The inertial turn counter persists in the chat's JSONB data.

Includes any pending reflection flags (from `flag_for_reflection` tool calls during the turn).

## Approach lights (disabled)

### `check_approach_threshold() -> str | None`

Currently returns `None`. Was designed to fire yellow (65%) and red (75%) context pressure warnings, but thresholds were calibrated for 200K and need recalibration for the 1M window.

## Module-level functions

### `generate_chat_id() -> str`

Generate a URL-safe chat ID. 12 chars, ~72 bits of entropy. Uses `secrets.token_urlsafe(9)`.

### `find_circadian_chat(chats, *, dawn_hour=6, now=None) -> Chat | None`

Find the most recent non-solitude chat from the current circadian day. The circadian day runs from `dawn_hour` to `dawn_hour` (default 6 AM to 6 AM), not midnight to midnight.
